大纲

如何将磁盘文件中的数据读写到内存中，或者反之？

  1. File:用于定位文件(对文件/文件夹本身操作，但是不能读写内容)
  2. IO:用于读写内容

File类概述

File类在包java.io.File下，代表操作系统的文件对象（文件，文件夹）
File类提供了：定位文件，获取文件本身信息，删除文件，创建文件（文件夹）等功能

File类API

File类可以定位文件/文件夹
File类封装的对象可以是存在的也可以是不存在的

File的创建删除API

File的查询API

File遍历API

注意事项:

  1. 当调用者不存在时，返回null
  2. 当调用者是一个文件时，返回null
  3. 当调用者是一个空文件夹时，返回一个长度为0的数组
  4. 当调用者是一个有内容的文件夹时，将里面所有的文件和文件夹的路径放在File数组中返回
  5. 当调用者是一个有隐藏内容时，将里面所有的包含隐藏内容都放在数组中返回

字符集

ASCII编码

一个字节，可以表示128个字符，英文数字都可以表示

GBK编码

2个字节，可以表示6万多字符，大部分中文都可以表示

Unicode万国码

utf-8，utf-16,等等，常用utf-8
3个字节，可以表示世界所有语言,程序猿都应该使用utf-8
编码前和编码后需要保持一致，否则乱码

编解码操作

IO流

  • I表示input，是数据从硬盘文件读入到内存，输入，负责读
  • O表示ouput，是内存程序的数据从内存到写出到硬盘文件的过程，输出，负责写

字节流的使用

用于文件拷贝

文件字节输入流

FileInputStream

注意事项：

  1. 字节输入流是一个字节一个字节的传输，而中文的2种编码容易引起在一个中文字符中间断开，导致乱码
  2. 字节输入流不适合读字符，适合读取整个文件
  3. 不是不可以读中午字符,可以将字节数不大的文件整体读取出来，使用readAllBytes()

文件字节输出流

注意事项：

  1. 写数据必须刷新数据,因为stream和磁盘文件中间有缓冲区buffer的存在，数据可能还在buffer中
  2. 释放资源包含刷新
  3. write(‘徐’)这种语法上是可以的，write(int)，但是会乱码，默认将字节组的第一个字节写入
