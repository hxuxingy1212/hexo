java规范

命名

驼峰明名

类名在驼峰的基础上首字母大写

不能以下划线或美元符号开始和结束

不难拼音和英文混合的方式，也不能中文

实体类中布尔变量不能is_开头，部分框架容易引起序列化错误

包名统一小写，点分隔符之间有且只有一个自然语义的英语单词

抽象类命名使用Abstract或者Base开头

异常类命名使用Exception结尾

测试类命名以测试的名称开始test结尾

数组用String[] a 表示，不能用String a[]

命名不能缩写，容易望文不生义

枚举类名要带Enum后缀,枚举成员名称需要全大写,单词用_隔开

常量定义

常量全部大写，单词下划线隔开

不允许没有定义的常量直接出现在代码中
比如： String key = “id:jump_”+tradeId;
当代码复制时容易漏掉下划线，导致缓存击穿

Long和long 赋值时要跟L不能是l

常量需要分开放

代码格式

大括号内空，则写成{},不为空则:左大括号前不换行,左大括号后换行,右大括号前换行,右大括号后还有else等代码则不换行，表示终止的右大括号后必须换行

左小括号和字符之间不出现空格，右小括号和字符之间也不出现空格

保留字中间必须加空格
